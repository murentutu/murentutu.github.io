  1. 执行hexo d命令时出现以下报错：


    解决方案如下：第二条设置系统代理，根据自身需求开启（如果开启了代理，就设置系统代理；未开启系统代理，则取消代理），并设置代理服务器端口为8890（因为开启代理并且代理服务器端口号为8890）；或者设置github邮箱隐私关闭解决hexo报错spwan failed - CairBin - 博客园。

  2. 配置主题是遇到评论功能依赖giscus实现，主题内没有详细介绍配置方法，结合网友实现：

    1. 公开仓库：设置评论的仓库需要设置为public；

    2. 安装giscus app：安装giscus app，直接点击安装后进行配置configure：

    3. 在仓库启用Discussions：在仓库设置中开启discussions：

      并在Features栏目里找到Discussions勾选。

    4. 配置代码：在giscus中根据自己需求填写内容，自动生成对应代码。

      将代码里的内容填入主题_config.yaml中的comment配置中及设置好。
